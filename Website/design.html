System Requirements

  1. Mandatory Requirements
    1. Functional
      2. In order for the FielDroid to succeed, the system shall:
      1. Perform 100% of autonomous tasks by itself
      2. Paint 100% of the line markings according to league standard regulations
      3. Plan a path within 1 minute given user inputs
      4. Stop within 2 seconds of receiving an emergency stop command
      5. Stop if communication with the command station times out for 10 seconds
    2. Non-Functional
      3. In order for the FielDroid to succeed, the system shall:
      1. Have a 30 cm error deviation between paint jobs
      2. Have an accuracy of 30 cm while performing paint jobs
      3. Have a set-up time of 5 minutes
      4. Spend at most 2 hours to paint a field
      5. Be capable of operating for 2 hours without replenishing resources
      6. Paint without physically damaging the field
  2. Desirable Requirements
    1. Functional
      2. Ideally, the FielDroid system shall:
      1. Monitor consumption of resources
      2. Plan the most efficient path within 2 minutes
      3. Report the robot's current status to the user once every 5 seconds
      4. Stop if an obstacle is detected within 1 meter
      5. Paint complex graphic designs
      6. Paint in multiple colors
      7. Move around obstacles detected along its path
      8. Relocalize itself if it gets lost while painting
    2. Non-Functional
      3. Ideally, the FielDroid system shall:
      1. Provide an affordable field-painting solution under $4,000
      2. Be operable using a mobile device application
      3. Be operable by personnel of minimum working age in the US
      4. Report battery life, paint consumption, percentage of task completion, and current state to the mobile device application
      5. Be able to paint other varieties of sport fields
      6. Require only 1 person to handle and operate

System descriptions

Localization

Localization is the most critical subsystem. Fieldroid uses a Leica robot total station to localize the robot in the environment. The Leica total station is initialized by calibrating to a 360 degree tracking prism mounted on top of the robot. Leica then continuously shoots a laser at the tracking prism to give an accurate pose of the robot in the world frame. The Leica also sends the timestamp, which can be used for clock synchronization. The data through the survey-gun is sent at around 1Hz, which requires that the system utilize dead-reckoning techniques for the interim. The data obtained from the Leica is within millimeter accuracy and therefore is highly reliable for precise and accurate field painting. This data can be integrated with the encoder data and in the future, the IMU data to present a multilayered pose in the world frame.







Drive

The drive system uses a robot-base provided by Sensible Machines. It is a robust drive system designed to work on fields. It has two drive wheels and four castor wheels for ease of maneuverability. It uses highly powerful DC motors and motor encoders to calculate the robot velocity and distance traveled. The RoboteQ motor controller processes the data from the encoders and communicates with the on-board BeagleBone which also receives the localization data from the survey-gun to correctly the position of the robot in the world frame.

The drive system and localization system work in strict synchronization to localize the robot in real time with the pre-calculated path planning data.

Communication Subsystem

For the communication system 3DR radios were utilized. The pair communicates at a frequency of 915 MHz and at a baud rate of 56700. The radios have been configured for single way communication. The transmitter radio is at the command station and receives the data from the laptop connected to the tracking station. The receiver radio receives the data and sends it to the BeagleBone. There are LEDs on both the radios which turns solid green when both radios establish communication and the LEDs blink when the connection is lost indicating fault. Also, there are LEDs which turn red and orange indicating data transfer over radios. The only issues seen with this component of the subsystem was when another team used the same pair of radios for communication; however, this problem can be resolved by giving separate id’s to the radios. These highly reliable radios ensure no loss of data and are otherwise a perfect choice for on-field communication.

Power System

Fieldroid has two separate power system modules. The first one is used to control the on-board electronics. Power is derived from 14.8V and 6.6 mAH Li-ion batteries. These batteries are rechargeable and provide sufficient current to meet the system requirements. The power is distributed to the unit modules using a custom designed power distribution printed circuit board (PCB). The PCB has built in protection circuitries for current spikes, overvoltage protection for improper fluctuations and noise rectification in case of disturbances. There were a few glitches in the PCB. These errors are on the schedule to be rectified in the spring semester as part of the risk mitigation plan. The ground traces on the PCB are thinner than the power traces which are undesirable. Also, the DC-DC converter pad layout is mirrored upside down, which causes the first issue.

The second power system is used to power the drive system. The batteries used for the drive system supply 24V at 50A. This is required to run the RoboteQ motor controller used for the drive system and the DC motors. The power system has appropriate LED indication at various points to indicate a functional system and give us warning in case of failure at any node. This is critical for easy debugging and robust system functionality.

Paint System

Our paint system was deconstructed from pre-built field marking equipment. It, therefore paints highly reliable and consistent field lines. It consists of a 6 gallon paint can, which can store sufficient paint to complete the field. It has a pump which triggers paint when receiving commands from the BeagleBone. It works on 12V and draws a maximum of 3.3A. This makes it highly power efficient which is desirable for our robot system. The robot has a spray mechanism at the bottom which is used to paint the field lines. It can be adjusted to change the width of the paint lines and also the height is adjustable to meet the user needs.












Control Systems

The robot’s control system is built upon a mathematical model that incorporates the encoders. This system uses a mathematical model that takes the encoder values and converts them into an X, Y, and Θ in the world frame. The robot can then control itself in the world frame based on goal points that are along the field path. The path of the robot takes into account the x,y, and theta of the robot in the world frame. The choice of large sweeping turns eliminates the problems associated with point turns. Damaging the grass and misaligned corners are likely if point turns were to be used in this context.

To control the robot, there needs to be in place a model to take the error in the world frame and apply it as control signals to the wheels. The RoboteQ motor controller allows power commands to be sent to the wheels, therefore, for a goal position, the power directions would look as depicted.

Tuning the control parameters presents difficulties on highly inertial systems. In order to make more sense of what could be controlled, the error in the robot’s frame was graphed over the path in order to analyze the issues with different sections of the path. This was the general tuning strategy for the overall system.

Safety System

To ensure the operator and equipment safety, a highly reliable emergency stop was incorporated. The emergency stop, if triggered, immediately halts the drive system. This meets MFR 2 which states that the robot shall stop within 2 seconds of receiving the emergency command. The E-stop makes the robot system highly robust and reliable to use for any user as well as the robot.
